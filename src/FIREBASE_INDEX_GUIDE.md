# 🔥 Firebase 인덱스 설정 가이드

## 📋 필요한 인덱스

최적화된 쿼리가 제대로 동작하려면 Firebase에 인덱스를 설정해야 합니다.

---

## 1. Firebase Console 접속

1. https://console.firebase.google.com 접속
2. 프로젝트 선택 (customer-pl)
3. **Firestore Database** 클릭
4. 상단 **색인** 탭 클릭

---

## 2. 인덱스 추가

### ✅ events 컬렉션 인덱스 (필수!)

| 필드 | 정렬 순서 |
|------|----------|
| date | 오름차순 |

**추가 방법:**
1. [+ 색인 추가] 클릭
2. 컬렉션 ID: `events`
3. 필드 경로: `date`
4. 정렬 순서: 오름차순
5. [만들기] 클릭

---

### 📌 복합 인덱스 (선택)

더 빠른 조회가 필요하면:

**events 컬렉션:**
| 필드 | 정렬 순서 |
|------|----------|
| staffId | 오름차순 |
| date | 오름차순 |

**customers 컬렉션:**
| 필드 | 정렬 순서 |
|------|----------|
| staffName | 오름차순 |
| custStatus | 오름차순 |

---

## 3. 자동 인덱스 생성

Firebase가 인덱스가 필요한 쿼리를 감지하면 Console에 에러 링크가 나타납니다.

```
FAILED_PRECONDITION: The query requires an index.
You can create it here: https://console.firebase.google.com/...
```

**이 링크를 클릭하면 자동으로 인덱스 생성 화면으로 이동합니다!**

---

## 4. 인덱스 상태 확인

- ✅ **사용 설정됨**: 정상
- 🔄 **빌드 중**: 잠시 대기 (1~5분)
- ❌ **오류**: 다시 생성

---

## 📊 최적화 효과

| 상황 | 인덱스 전 | 인덱스 후 |
|------|----------|----------|
| 이벤트 조회 | 2~5초 | 0.3~0.5초 |
| 월 변경 | 느림 | 빠름 |

---

## ⚠️ 주의사항

- 인덱스 생성에 1~5분 소요됩니다
- 무료 플랜은 인덱스 제한이 있습니다 (200개)
- 필요한 인덱스만 추가하세요

---

## 📞 문제 발생 시

Console에 에러 메시지가 나오면 링크를 클릭해서 인덱스를 자동 생성하면 됩니다!
